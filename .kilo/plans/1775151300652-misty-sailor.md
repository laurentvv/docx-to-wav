## Phase 1 : Corrections Critiques (Bugs)

### 1.1 Corriger le bug du tuple non déballé dans `convert_docx.py`
- **Fichier** : `convert_docx.py`
- **Lignes** : 36-49
- **Problème** : `extract_text_from_docx()` retourne des tuples `(text, is_title)` mais le code itère comme si c'était des strings
- **Solution** : Déballer le tuple correctement
```python
# Avant
paragraphs = extract_text_from_docx(docx_path)
for i, paragraph in enumerate(paragraphs):
    generator = pipeline(paragraph, ...)

# Après
chunks = extract_text_from_docx(docx_path)
for i, (paragraph, is_title) in enumerate(chunks):
    # Optionnel : traiter les titres différemment
    generator = pipeline(paragraph, ...)
```

### 1.2 Corriger la version Python dans `pyproject.toml`
- **Fichier** : `pyproject.toml`
- **Ligne** : 6
- **Problème** : `requires-python = ">=3.14"` (version inexistante)
- **Solution** : `requires-python = ">=3.12"`

## Phase 3 : Corrections de Robustesse

### 3.1 Écriture atomique du fichier MP3
- **Fichier** : `convert_docx_google_tts.py`
- **Lignes** : 90-102
- **Solution** : Écrire dans un fichier temporaire, puis renommer
```python
import tempfile
import shutil

temp_path = f"{mp3_path}.tmp"
try:
    with open(temp_path, "wb") as out_file:
        for batch in batches:
            audio_content = synthesize_ssml(...)
            out_file.write(audio_content)
    shutil.move(temp_path, mp3_path)
except Exception as e:
    if os.path.exists(temp_path):
        os.remove(temp_path)
    raise
```
