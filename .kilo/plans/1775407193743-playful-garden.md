# Plan : Correction de la gestion des apostrophes françaises dans le TTS Google

## Problème identifié

Le mot "c'était" est mal prononcé par le modèle TTS Google Chirp3-HD.

### Causes dans le code actuel

1. **Code mort dans `convert_docx_google_tts.py` (lignes 61-70)** :
   - `fixed_text` est calculé mais jamais utilisé
   - La correction est réappliquée ensuite sur `safe_text`

2. **Regex insuffisant dans `utils.py` (ligne 67)** :
   - `\w` en Python standard ne capture pas toujours les caractères accentués français (`é`, `è`, `ê`, etc.)
   - Le pattern `[{voyelles}][\w]*` peut échouer sur des mots comme "était"

## Solution proposée

### 1. Nettoyer `convert_docx_google_tts.py`

Supprimer la variable `fixed_text` inutilisée et simplifier la logique :

```python
for text, is_title in chunks:
    # Échapper les caractères HTML mais garder l'apostrophe
    safe_text = html.escape(text).replace("&#x27;", "'").replace("&quot;", '"')
    processed_text = fix_french_elisions_ssml(safe_text)
    
    if is_title:
        formatted_text = f'<prosody rate="0.9"><emphasis level="strong">{processed_text}</emphasis></prosody> <break time="1.5s"/>'
    else:
        formatted_text = processed_text
```

### 2. Améliorer `fix_french_elisions_ssml` dans `utils.py`

Remplacer `\w` par une classe de caractères explicite incluant les accents français :

```python
def fix_french_elisions_ssml(text: str) -> str:
    """
    Corrige les erreurs de prononciation de Chirp3-HD sur les apostrophes.
    """
    if not text: return ""
    
    # Caractères français complets (lettres + accents)
    lettres_fr = r"a-zA-ZàâäéèêëïîôùûüÿçÀÂÄÉÈÊËÏÎÔÙÛÜŸÇ"
    
    # Cas général : élisions suivies d'un mot commençant par une voyelle
    text = re.sub(rf"\b([ncjlsmdt])'([{lettres_fr}]+)", 
                  r"<sub alias='\1 \2'>\1'\2</sub>", text)
    
    # Cas spécifique pour "qu'" -> prononciation "k"
    text = re.sub(rf"\b(qu)'([{lettres_fr}]+)", 
                  r"<sub alias='k \2'>qu'\2</sub>", text)
    
    return text
```

**Note importante** : L'alias inclut maintenant un espace (`\1 \2` au lieu de `\1\2`) pour forcer la séparation syllabique et améliorer la prononciation.
